# 状态机说明书（初稿）

文档名称：内部培训考试系统状态机说明书
版本号：v0.1
日期：2026-03-09

------

## 1. 设计原则

1. 状态必须存库
2. 状态变更只能走服务层
3. 非法迁移必须拒绝
4. 关键迁移必须记录审计日志
5. 审核、发布、考试状态迁移必须带操作人

------

# 2. 文档状态机

## 2.1 状态集合

- `uploaded`
- `parsing`
- `parsed`
- `failed`
- `archived`

## 2.2 状态迁移

### uploaded → parsing

触发条件：

- 文档上传成功
- 系统自动创建解析任务

触发角色：

- 系统

### parsing → parsed

触发条件：

- 解析任务成功完成
- 至少生成有效解析结果

触发角色：

- 系统 Worker

### parsing → failed

触发条件：

- 解析失败
- OCR 失败且无可用文本结果

### failed → parsing

触发条件：

- 用户发起重试

触发角色：

- 培训专员 / 管理员

### parsed → archived

触发条件：

- 文档归档

触发角色：

- 培训专员 / 管理员

------

# 3. 候选知识点状态机

## 3.1 状态集合

- `pending`
- `accepted`
- `ignored`
- `merged`

## 3.2 迁移规则

### pending → accepted

条件：

- 培训专员确认候选项有效
- 写入正式 knowledge_points

### pending → ignored

条件：

- 培训专员认为无效

### pending → merged

条件：

- 候选项被合并到已有知识点

------

# 4. 正式知识点状态机

## 4.1 状态集合

- `draft`
- `active`
- `inactive`
- `merged`

## 4.2 迁移规则

### draft → active

条件：

- 通过人工确认并启用

### active → inactive

条件：

- 不再使用但保留引用历史

### inactive → active

条件：

- 重新启用

### active / inactive → merged

条件：

- 被合并至其他知识点

------

# 5. 课程版本状态机

## 5.1 状态集合

- `draft`
- `ai_generated`
- `pending_review`
- `in_review`
- `approved`
- `rejected`
- `published`
- `archived`

## 5.2 迁移规则

### draft → pending_review

条件：

- 培训专员提交审核

### ai_generated → draft

条件：

- 培训专员手工编辑保存
- 进入人工可编辑草稿态

### ai_generated → pending_review

条件：

- 不修改直接提交审核

### pending_review → in_review

条件：

- 审核任务被接单或开始处理

触发角色：

- 审核专家

### in_review → approved

条件：

- 初审/复审通过
- 满足当前审核阶段完成条件

### in_review → rejected

条件：

- 审核驳回

### rejected → draft

条件：

- 培训专员基于驳回意见修改后重新保存为草稿

### approved → published

条件：

- 培训专员执行发布
- 生成 publish_record

### published → archived

条件：

- 被新版本替代或停用

### published → draft

不允许直接迁移
规则：

- 必须复制为新版本，再编辑

------

# 6. 题目版本状态机

与课程版本基本一致，状态同下：

- `draft`
- `ai_generated`
- `pending_review`
- `in_review`
- `approved`
- `rejected`
- `published`
- `archived`

特殊规则：

### published → draft

不允许直接修改
必须：

- 复制新版本
- 新版本进入 draft

------

# 7. 审核任务状态机

## 7.1 状态集合

- `pending`
- `processing`
- `approved`
- `rejected`
- `returned`

## 7.2 迁移规则

### pending → processing

条件：

- 审核专家打开并开始处理

### processing → approved

条件：

- 审核通过

### processing → rejected

条件：

- 审核不通过，直接驳回版本

### processing → returned

条件：

- 需修改后再提，不直接判死

说明：

- `returned` 与 `rejected` 的业务语义可保留差异
- 一期实现时也可合并为驳回，但数据层建议保留

------

# 8. 发布记录状态机

## 8.1 状态集合

- `scheduled`
- `published`
- `rolled_back`
- `cancelled`

## 8.2 迁移规则

### scheduled → published

条件：

- 到达生效时间或立即发布成功

### scheduled → cancelled

条件：

- 生效前取消

### published → rolled_back

条件：

- 执行回滚操作

说明：

- 回滚后应生成新的 publish_record 或标记旧记录 rolled_back
- 推荐两者都做：旧记录标记 rolled_back，新记录表示恢复到上版

------

# 9. 培训任务状态机

## 9.1 状态集合

- `draft`
- `published`
- `ongoing`
- `completed`
- `cancelled`

## 9.2 迁移规则

### draft → published

条件：

- 培训专员发布任务
- 分配对象已确定
- 截止时间有效

### published → ongoing

条件：

- 任务到达开始执行状态
- 用户开始学习或系统到达生效时间

### ongoing → completed

条件：

- 超过截止时间，或全部对象完成

### draft / published / ongoing → cancelled

条件：

- 管理员或培训专员取消任务

限制：

- 已有考试尝试数据时，取消需二次确认并留审计日志

------

# 10. 培训分配状态机

## 10.1 状态集合

- `pending`
- `in_progress`
- `completed`
- `overdue`

## 10.2 迁移规则

### pending → in_progress

条件：

- 用户开始学习或开始考试

### in_progress → completed

条件：

- 学习完成且考试完成（如配置考试）
- 或仅学习任务完成（无考试）

### pending / in_progress → overdue

条件：

- 超过截止时间仍未完成

### overdue → completed

条件：

- 补做完成且业务允许

------

# 11. 考试状态机

## 11.1 exams 状态集合

- `scheduled`
- `ongoing`
- `submitted`
- `auto_graded`
- `manual_grading`
- `finished`
- `cancelled`

说明：
这里更准确地说是“考试执行态/成绩态”，对运营对象有用。

## 11.2 迁移规则

### scheduled → ongoing

条件：

- 到达考试开始时间或用户开始考试

### ongoing → submitted

条件：

- 用户交卷或时间到自动交卷

### submitted → auto_graded

条件：

- 客观题自动判分完成

### auto_graded → manual_grading

条件：

- 含主观题待人工评分

### auto_graded → finished

条件：

- 全部题型已判分完成

### manual_grading → finished

条件：

- 人工评分完成，成绩发布

### scheduled / ongoing → cancelled

条件：

- 考试取消

------

# 12. 考试作答会话状态机

这个更关键，真正控制答卷生命周期的是 `exam_attempts`。

## 12.1 状态集合

- `scheduled`
- `ongoing`
- `submitted`
- `auto_graded`
- `manual_grading`
- `finished`
- `cancelled`

## 12.2 迁移规则

### scheduled → ongoing

条件：

- 用户点击开始考试
- 创建开始时间

### ongoing → submitted

条件：

- 用户主动交卷
- 或考试时间耗尽系统自动交卷

### submitted → auto_graded

条件：

- 客观题评分完成

### auto_graded → manual_grading

条件：

- 存在主观题待评

### auto_graded → finished

条件：

- 无主观题或主观题不纳入本次成绩

### manual_grading → finished

条件：

- 人工评分结束

### ongoing → cancelled

条件：

- 考试被系统取消
- 或管理员判定作废

------

# 13. 异步任务状态机

## 13.1 状态集合

- `pending`
- `processing`
- `succeeded`
- `failed`
- `cancelled`

## 13.2 迁移规则

### pending → processing

条件：

- Worker 消费任务

### processing → succeeded

条件：

- 任务完成且结果通过校验

### processing → failed

条件：

- 执行失败
- 输出不合法
- 超时达到上限

### failed → pending

条件：

- 允许重试且用户或系统发起重试

### pending / processing → cancelled

条件：

- 任务不再执行，且未产生不可逆业务副作用

------

# 14. 特殊状态规则

## 14.1 已发布对象不可原地编辑

适用：

- 课程版本
- 题目版本

规则：

- published 状态不可直接修改内容
- 编辑必须复制新版本
- 新版本从 draft 开始重新走审核

------

## 14.2 审核中对象不可覆盖

适用：

- pending_review / in_review

规则：

- 不允许直接覆盖待审版本
- 修改必须新建草稿版本
- 原待审版本继续保留审核上下文

------

## 14.3 考试提交幂等

适用：

- exam_attempts

规则：

- submitted 及之后状态不允许再次提交产生第二份正式结果
- 重复请求返回首个有效提交结果

------

## 14.4 失败任务可重试，但要留痕

适用：

- 文档解析
- 知识点抽取
- 课程生成
- 题目生成

规则：

- 每次重试增加 retry_count
- 不能覆盖历史失败原因

------

# 15. 状态迁移审计要求

下列动作必须写入 `audit_logs`：

1. 文档重试解析
2. 知识点合并/停用
3. 课程提交审核
4. 题目提交审核
5. 审核通过/驳回/退回
6. 内容发布
7. 内容回滚
8. 培训任务发布/取消
9. 考试开始/提交/作废
10. 成绩发布
