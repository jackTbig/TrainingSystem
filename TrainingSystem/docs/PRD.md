# 2. 产品概述

## 2.1 产品背景

当前企业内部培训高度依赖人工整理资料、人工编写课程、人工出题和人工组织考试，存在以下问题：

1. 培训资料分散，知识难沉淀
2. 课程制作效率低，更新滞后
3. 题库建设成本高，覆盖不足
4. 内容审核缺乏结构化流程
5. 培训发布、学习、考试、成绩分析链路割裂
6. 难以形成“知识资产 → 培训内容 → 考试评估”的闭环

因此需要建设一套内部培训考试系统，完成以下闭环：

**企业文档入库 → 知识点抽取 → AI辅助生成课程与试题 → 人工审核 → 发布培训任务 → 学员学习与考试 → 结果沉淀与分析**

------

## 2.2 产品定位

本系统定位为：

**企业内部知识驱动的培训与考试平台**

系统核心不是单纯的 LMS，也不是单纯的 AI 内容生成工具，而是一个包含以下三类能力的平台：

1. **知识资产管理能力**：沉淀内部培训知识库
2. **内容生产能力**：AI 辅助生成课程和题目，并进入审核流程
3. **培训考试运营能力**：完成发布、学习、考试、成绩与追踪闭环

------

## 2.3 一期产品目标（P0）

一期目标聚焦“从文档到培训考试闭环”的最小可落地版本，必须实现：

1. 企业培训文档统一上传、解析和检索
2. 基于文档/知识点生成课程草稿
3. 基于知识点生成客观题与主观题草稿
4. 人工审核课程与试题并形成发布版本
5. 发布培训任务给指定员工/部门
6. 支持员工在线学习、参加考试、查看成绩
7. 支持管理员查看任务完成情况、考试结果和操作审计

------

## 2.4 一期不做范围（Out of Scope）

以下内容不纳入一期正式交付范围，仅保留设计接口或扩展点：

1. 多租户能力
2. 单点登录（SSO / LDAP / OAuth 企业统一身份）
3. 高级知识图谱可视化
4. 视频课程直播 / 录播
5. 主观题自动评分正式上线
6. 高级防作弊能力（摄像头监考、行为分析、录屏留证）
7. 移动端 App 原生开发
8. 复杂 BPM 审批引擎
9. 组织外部用户接入
10. 多语言界面

# 3. 成功标准

## 3.1 业务指标

一期上线 3 个月后，目标如下：

1. 课程制作效率提升 ≥ 50%
2. 客观题生成后可进入审核的比例 ≥ 80%
3. 课程草稿经一次人工审核后可发布的比例 ≥ 70%
4. 培训任务按期完成率 ≥ 85%
5. 考试成绩自动判分覆盖率 ≥ 80%（仅客观题）
6. 培训资料集中化入库率 ≥ 90%

------

## 3.2 系统指标

1. 常规页面加载时间 < 3 秒
2. 常规 API 响应时间 < 500ms（异步任务接口除外）
3. 文档上传成功率 ≥ 99%
4. AI 生成任务可追踪率 = 100%
5. 审核与发布全链路审计可追溯率 = 100%

------

# 4. 用户角色与权限边界

## 4.1 角色定义

### 4.1.1 系统管理员

负责系统配置、组织管理、用户权限管理、全局监控和审计。

### 4.1.2 培训专员

负责文档上传、知识点维护、发起课程/题目生成、编辑内容、提交审核、发布培训任务。

### 4.1.3 审核专家

负责审核课程和题目，给出修改意见、通过或驳回审核。

### 4.1.4 普通员工

负责接收培训任务、学习课程、参加考试、查看成绩。

------

## 4.2 权限原则

一期采用 **RBAC（基于角色的访问控制）**，后续可扩展到角色 + 数据范围。

权限原则如下：

1. 系统管理员拥有全局配置与全局数据可见权限
2. 培训专员可管理自己有权限范围内的培训内容
3. 审核专家只能处理分配给自己的审核任务
4. 普通员工只能访问分配给自己的培训任务、考试、成绩
5. 所有“发布”“驳回”“回滚”操作必须记录审计日志

------

# 5. 一期业务范围

一期功能划分为 8 个业务模块：

1. 用户与权限
2. 文档知识库
3. 知识点管理
4. 课程生成与审核
5. 题库生成与审核
6. 发布与培训任务
7. 学员学习与考试
8. 统计与审计

------

# 6. 业务流程总览

## 6.1 主流程

**文档上传 → 文档解析 → 知识点抽取/维护 → 课程/试题生成 → 人工审核 → 发布版本 → 培训任务下发 → 学员学习 → 考试 → 成绩与分析**

------

## 6.2 子流程一：知识入库

1. 培训专员上传文档
2. 系统创建文档记录并异步解析
3. 解析完成后生成文档结构、文本分块、元数据
4. 系统抽取候选知识点
5. 培训专员人工确认和调整知识点
6. 知识点进入可引用状态

------

## 6.3 子流程二：课程生产

1. 培训专员选择知识点或文档
2. 发起课程生成任务
3. AI 生成课程结构化草稿
4. 培训专员编辑后提交审核
5. 审核专家审核通过 / 驳回
6. 通过后形成可发布版本

------

## 6.4 子流程三：题库生产

1. 培训专员选择知识点
2. 发起题目生成任务
3. AI 生成结构化题目草稿
4. 培训专员预编辑后提交审核
5. 审核专家通过 / 驳回
6. 通过后进入可组卷题库

------

## 6.5 子流程四：培训执行

1. 培训专员发布培训任务
2. 指定人员 / 部门 / 截止日期 / 补考规则
3. 员工接收任务并学习课程
4. 员工参加考试
5. 系统自动判分客观题
6. 成绩发布，培训专员查看完成情况

------

# 7. 核心业务规则

## 7.1 文档处理规则

1. 支持格式：DOCX、PDF、PPTX、XLSX、TXT
2. 单文件大小上限：100MB
3. 单次批量上传上限：20 个文件
4. 上传成功后必须生成唯一文档记录
5. 解析采用异步方式执行
6. 文档解析失败必须可重试
7. OCR 仅作为扫描类 PDF 的兜底能力，不保证复杂排版 100% 还原
8. 一期只要求“文本可抽取、结构可识别、块级定位可记录”，不要求复杂图表语义理解

------

## 7.2 知识点规则

1. 知识点为最小可训练单元
2. 知识点必须可被人工新增、修改、合并、停用
3. 一个知识点可关联多个文档片段
4. 一个文档片段可关联多个知识点
5. 一期仅支持“知识点 + 关联关系”，不实现复杂图数据库
6. 知识点关系至少支持：
   - 上下级
   - 前置依赖
   - 相关知识点

------

## 7.3 课程规则

1. 课程必须基于文档或知识点生成
2. 课程生成结果必须为结构化草稿，而不是纯文本
3. 每门课程默认 5-10 章，可由培训专员调整
4. 每章默认 3-5 个核心知识点
5. AI 可生成案例，但必须标记来源类型：
   - document：来源于现有文档材料
   - generated：模型生成案例
6. 发布前必须经过人工审核
7. 已发布课程修改后必须形成新版本，不允许直接覆盖线上版本

------

## 7.4 题目规则

1. 题目必须关联至少一个知识点
2. 一期支持题型：
   - 单选题
   - 多选题
   - 判断题
   - 填空题
   - 简答题
   - 案例分析题
3. 自动判分范围：
   - 单选、多选、判断、填空：系统自动判分
   - 简答、案例分析：一期默认人工评阅或暂不纳入正式成绩
4. 每个知识点默认生成不少于 3 道题
5. 题目草稿必须进入审核流程
6. 发布入库后题目必须可被组卷引用
7. 已发布题目修改后必须产生新版本

------

## 7.5 审核规则

1. 审核对象包括课程和题目
2. 一期采用二级审核模型：
   - 初审
   - 复审
3. 驳回后对象返回“待修改”状态
4. 审核意见必须留痕
5. 审核通过后才可进入发布状态
6. 审核过程必须支持版本对比

------

## 7.6 发布规则

1. 只有“审核通过”的课程和题目可以发布
2. 发布支持指定生效时间
3. 发布范围支持：
   - 指定用户
   - 指定部门
4. 已发布内容必须可查看发布记录
5. 发布版本可回滚到上一个已发布版本
6. 回滚必须记录操作人、原因和时间

------

## 7.7 培训任务规则

1. 培训任务必须绑定至少一门课程
2. 培训任务可选择是否包含考试
3. 培训任务必须设定截止时间
4. 可设置是否允许补考
5. 学习进度必须自动记录
6. 考试成绩必须与培训任务关联

------

## 7.8 考试规则

1. 一期支持固定试卷与规则组卷两种模式
2. 考试必须记录开始时间、提交时间、耗时
3. 考试过程支持倒计时
4. 支持前端切屏事件记录，但一期仅做事件留痕，不做强制封禁
5. 客观题交卷后自动判分
6. 断网重连后应恢复未提交答卷状态
7. 交卷接口必须幂等，防止重复交卷

------

# 8. 状态机定义

## 8.1 文档状态

- uploaded：已上传
- parsing：解析中
- parsed：解析成功
- failed：解析失败
- archived：已归档

状态规则：

1. uploaded 后自动进入 parsing
2. parsing 成功进入 parsed
3. parsing 失败进入 failed
4. failed 可重试回到 parsing
5. archived 为人工归档终态

------

## 8.2 知识点状态

- draft：草稿
- active：启用
- inactive：停用
- merged：已合并

------

## 8.3 课程状态

- draft：草稿
- ai_generated：AI 已生成
- pending_review：待审核
- in_review：审核中
- approved：审核通过
- rejected：已驳回
- published：已发布
- archived：已归档

------

## 8.4 题目状态

- draft
- ai_generated
- pending_review
- in_review
- approved
- rejected
- published
- archived

------

## 8.5 审核任务状态

- pending：待处理
- processing：处理中
- approved：通过
- rejected：驳回
- returned：退回修改

------

## 8.6 培训任务状态

- draft：草稿
- published：已发布
- ongoing：进行中
- completed：已结束
- cancelled：已取消

------

## 8.7 考试状态

- scheduled：待开始
- ongoing：进行中
- submitted：已提交
- auto_graded：已自动判分
- manual_grading：待人工评阅
- finished：已完成
- cancelled：已取消

# 10. 功能需求

------

## 10.1 用户与权限模块

### F0.1 用户管理

支持管理员维护用户基本信息、启用/停用状态、所属部门和角色。

验收标准：

- 支持新增、编辑、停用用户
- 支持按部门查询用户
- 支持用户角色配置
- 停用用户后不可登录系统
- 用户所有关键操作可审计

------

### F0.2 角色权限管理

支持角色管理与菜单/接口权限配置。

验收标准：

- 预置管理员、培训专员、审核专家、普通员工四类角色
- 支持为角色配置菜单权限
- 支持接口访问鉴权
- 权限变更后立即生效或在重新登录后生效

------

## 10.2 文档知识库模块

### F1.1 文档上传

支持上传培训资料并创建文档记录。

验收标准：

- 支持 DOCX、PDF、PPTX、XLSX、TXT
- 单文件 ≤ 100MB
- 批量上传 ≤ 20 个
- 显示上传进度
- 支持断点续传
- 上传成功后生成 document_id

------

### F1.2 文档解析

系统异步解析文档文本结构和元数据。

输入：document_id
输出：文档解析结果

验收标准：

- 可识别章节标题层级
- 可抽取正文文本
- 可生成文档分块
- 支持扫描版 PDF OCR 兜底
- 记录解析状态和错误原因
- 支持失败重试

------

### F1.3 文档检索与预览

支持查看原始文档、解析文本及结构结果。

验收标准：

- 支持按标题、标签、上传人搜索
- 支持查看文档原文件
- 支持查看解析结果
- 支持查看文档块与知识点关联结果

------

## 10.3 知识点管理模块

### F2.1 候选知识点抽取

系统根据文档内容自动抽取候选知识点。

验收标准：

- 可展示候选知识点列表
- 可展示候选知识点来源文本
- 允许人工确认或忽略
- 支持批量确认

------

### F2.2 知识点维护

支持人工调整知识点体系。

验收标准：

- 支持新增、编辑、停用、合并知识点
- 支持设置上级知识点
- 支持设置关联关系
- 支持查看知识点引用情况
- 支持知识点搜索

------

## 10.4 课程生成与管理模块

### F3.1 课程生成

基于文档或知识点生成课程草稿。

输入：

- 文档 ID 列表或知识点 ID 列表
- 课程标题（可选）
- 生成配置

输出：

- 结构化课程草稿

默认生成配置：

- 章节数：5-10
- 每章知识点数：3-5
- 案例比例：20%-30%
- 每章可生成配套练习

验收标准：

- 生成结果包含课程标题、摘要、章节、知识点映射
- 每章必须有正文内容
- 章节顺序可调整
- 生成结果以草稿版本形式保存
- 生成任务状态可追踪

------

### F3.2 课程编辑

培训专员可在线修改课程草稿。

验收标准：

- 支持富文本/Markdown 编辑
- 支持章节增删改排序
- 支持调整关联知识点
- 支持查看来源文档引用
- 支持保存草稿
- 支持版本比较

------

### F3.3 课程提交审核

将课程草稿提交到审核流。

验收标准：

- 仅草稿或 ai_generated 状态可提交
- 提交后状态变为 pending_review
- 自动创建审核任务
- 提交行为记录审计日志

------

## 10.5 题库生成与管理模块

### F4.1 题目生成

根据知识点生成题目草稿。

输入：

- 知识点 ID 列表
- 题型配置
- 题量配置
- 难度分布配置

默认规则：

- 每个知识点不少于 3 道题
- 难度分布：易 30% / 中 50% / 难 20%

验收标准：

- 支持单选、多选、判断、填空、简答、案例分析
- 客观题必须生成标准答案

# 11. AI 处理链路要求

本节只定义产品级约束，详细实现放到架构文档。

## 11.1 文档入库链路

流程：

文档上传 → 存储原文件 → 解析文本与结构 → 文本分块 → 元数据抽取 → 建立检索索引

要求：

1. 所有步骤必须可追踪
2. 任一步失败必须记录失败原因
3. 支持重试
4. 解析结果必须持久化，不依赖临时缓存

------

## 11.2 知识点抽取链路

流程：

文档分块 → 候选知识点抽取 → 去重归并 → 人工确认 → 知识点入库

要求：

1. 候选知识点必须保留来源文本
2. 人工确认前不得直接进入正式知识点体系
3. 支持合并重复知识点

------

## 11.3 课程生成链路

流程：

选择知识点/文档 → 检索上下文 → 调用模型生成 → 输出结构化课程草稿 → 人工编辑 → 审核

要求：

1. 输出必须为结构化数据
2. 必须可追踪生成批次
3. 必须记录输入范围与生成配置
4. 必须支持重新生成

------

## 11.4 题目生成链路

流程：

选择知识点 → 调用模型生成题目 → 规则校验 → 相似题检测 → 保存题目草稿 → 人工审核

要求：

1. 输出必须为结构化题目数据
2. 客观题必须包含标准答案
3. 所有题目必须包含解析
4. 必须支持批量生成和批量审核

------

# 14. 异常与失败处理规则

这一节是为了避免后续架构和 agent 忽略边界条件。

## 14.1 文档异常

1. 上传失败：允许重新上传
2. 解析失败：记录原因并允许重试
3. OCR 失败：标记失败，不阻塞其他文档处理
4. 重复上传：通过文件 hash 识别并提示是否复用历史解析结果

------

## 14.2 AI 生成异常

1. 模型超时：任务失败并可重试
2. 输出格式非法：进入校验失败状态，不进入审核
3. 上下文不足：提示生成质量风险
4. 生成过程中断：保留失败记录与输入参数

------

## 14.3 审核异常

1. 审核中内容被修改：必须生成新草稿版本，不得直接覆盖待审版本
2. 审核任务超时未处理：可提醒或重新分配
3. 审核通过后再次修改：必须重新进入审核流程

------

## 14.4 考试异常

1. 网络中断：允许恢复未提交答卷
2. 重复交卷：接口幂等，只保留第一次有效提交
3. 自动判分失败：进入待人工处理状态
4. 切屏事件：只记录，不自动终止考试

------
